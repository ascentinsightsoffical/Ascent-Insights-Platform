# Shared Platform Assets

## Purpose

The shared module provides a future home for cross-platform assets that should be reused across the website, backend, admin dashboard, and AI initiatives.

## What Belongs Here

- Type definitions and contracts
- Shared constants and interfaces
- Design tokens and standards
- Utility references and shared documentation

## What Should Not Be Placed Here

This folder should not contain application-specific logic or environment-specific runtime configuration.

## Relationship to the Platform

Shared assets should be governed centrally so that common patterns remain consistent as the platform grows.
